          authenticate on behalf of users with scoped, time-limited access.
        </p>

        <h2>Why MCP for Authentication?</h2>
        <p>
          Traditional authentication is designed for human users, but AI agents need different 
          capabilities:
        </p>

          <li><strong>Delegated Access:</strong> Agents act on behalf of users</li>
          <li><strong>Scoped Permissions:</strong> Limited access to specific resources</li>
          <li><strong>Time-Limited:</strong> Short-lived tokens for security</li>
          <li><strong>Audit Trail:</strong> Track agent actions and access</li>
        </ul>

        <h2>Use Cases</h2>
        <p>
          MCP authentication enables AI agents to:
        </p>

        <ul>
          <li><strong>Debug Issues:</strong> Agents can access user data to diagnose problems</li>
          <li><strong>Automate Tasks:</strong> Perform routine operations on behalf of users</li>
          <li><strong>Data Analysis:</strong> Analyze user data with appropriate permissions</li>
          <li><strong>Support:</strong> Help users with account-related issues</li>
        </ul>

        <h2>Security Model</h2>
        <p>
          MCP authentication includes several security features:
        </p>

        <ul>
          <li><strong>Short-lived Tokens:</strong> Default 15-minute expiration</li>
          <li><strong>Scope-based Access:</strong> Agents can only access permitted resources</li>
          <li><strong>Token Revocation:</strong> Immediate invalidation when needed</li>
          <li><strong>Audit Logging:</strong> Track all agent actions</li>
        </ul>

  })

const { token } = await response.json();
